FinPay Deployment Setup (GitHub -> AWS)

This guide covers a simple, safe deployment flow for CI/CD and how to switch behavior automatically in production.

1. Pipeline Overview

  1. Push code to GitHub branch.
  2. CodePipeline triggers CodeBuild.
  3. CodeBuild runs tests and builds Docker image using buildspec.yml.
  4. Image is pushed to ECR.
  5. ECS deploy stage updates the service using imagedefinitions.json.

2. Secrets Source of Truth

Use AWS Secrets Manager or SSM Parameter Store for runtime secrets. Do not store production secrets in GitHub.

CodeBuild variables are for build steps. App runtime secrets must be configured on the ECS task definition.

3. Required Runtime Variables (ECS)

  • APP_ENV=production
  • STRIPE_SECRET_KEY
  • STRIPE_PUBLISHABLE_KEY
  • STRIPE_WEBHOOK_SECRET
  • API_ALLOWED_ORIGIN

Optional but recommended

  • DB_HOST
  • DB_PORT
  • DB_NAME
  • DB_USER
  • DB_PASS

4. Production Switch (If Statement)

In api/v1/lib/config.php, the system now checks APP_ENV. When set to production, local secrets file is ignored. 

$appEnv = strtolower((string)(getenv('APP_ENV') ?: 'development'));

if ($appEnv !== 'production') {
    // load config.local.php for local development only
}

5. Deployment Steps

  1. Create/update secrets in Secrets Manager (or SSM).
  2. Map those secrets to ECS task definition container environment.
  3. Set APP_ENV=production in ECS task definition.
  4. Push to GitHub branch connected to pipeline.
  5. Watch CodePipeline build and deploy stages.
  6. Validate app and webhook endpoint after deployment.

6. Verification Checklist

  • Deposit create endpoint returns JSON without parse errors.
  • Card deposit creates Stripe PaymentIntent in stripe mode.
  • Webhook processing marks card deposit as completed.
  • No secrets are present in git history or repo files.

Status

Ready for CI/CD: production secrets are runtime-injected and local files are ignored in production mode.

Do not commit real secrets to GitHub.

Rotate any previously exposed test or live keys before next deployment.
